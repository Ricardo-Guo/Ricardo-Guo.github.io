基于 OpenVINS 的无人机集群协同自定位系统

基于 ROS2 和 OpenVINS 实现无 GPS 环境下无人机集群高精度视觉惯性定位

Apr 24, 2024
English

🎯 项目概览

解决无人机集群在无 GPS 环境下的漂移问题。

本项目实现了一套综合的视觉惯性里程计（VIO）系统，结合了 OpenVINS、ROS2 通信架构 和 集群协同算法，在仿真和真实环境下都能实现高精度的无人机集群定位。

技术栈

  • 中间件: ROS2 (Humble/Foxy)
  • 编程语言: C++17/20, Python
  • VIO 引擎: OpenVINS
  • 仿真: Gazebo Classic, PX4 SITL
  • 飞控: PX4 Autopilot, MAVROS
  • 计算机视觉: OpenCV, Eigen
  • 优化库: Ceres Solver, g2o
  • 可视化: RViz2, PlotJuggler

  • 真实光照 和阴影渲染

挑战与解决方案

挑战 1: ROS2 跨设备通信延迟

问题: 网络延迟导致位姿更新延迟 50-100ms。

解决方案:

  • 实现基于 ZeroMQ 的高频位姿数据通道
  • 使用时间关键消息的 ROS2 QoS 策略
  • 实现 < 10ms 端到端延迟

// 用于低延迟位姿传输的自定义 ZeroMQ 发布器
auto publisher = zmq::socket_t(context, zmq::socket_style::pub);
publisher.bind("tcp://*:5555");

while (rclcpp::ok()) {
    PoseMessage msg = get_latest_pose();
    publisher.send(zmq::buffer(msg.serialize()), zmq::send_flags::none);
}

挑战 2: 震动下的 IMU 噪声

问题: 电机震动导致 IMU 尖峰，降低 VIO 精度。

解决方案:

  • 硬件: 添加减震支架
  • 软件: 自适应阈值异常值抑制
  • 滤波器: 带震动检测的互补滤波器

// 自适应 IMU 噪声模型
double get_adaptive_noise(double vibration_level) {
    return base_noise_ * (1.0 + vibration_gain_ * vibration_level);
}

// 异常值抑制
if (accel_magnitude > threshold) {
    reject_measurement();
    flag_vibration_detected();
}

挑战 3: 计算优化

问题: OpenVINS 处理每帧耗时 80ms，限制在 12Hz。

解决方案:

  • GPU 加速: 将特征跟踪移植到 CUDA
  • 多线程: 多机并行 MSCKF 更新
  • 代码优化: 矩阵运算的 SIMD 指令

性能提升：

优化 处理时间 频率
原始 80ms 12Hz
+ 多线程 45ms 22Hz
+ GPU 加速 25ms 40Hz
