OpenVINS-based UAV Swarm Cooperative Self-Localization System

Apr 24, 2024
中文

🎯 Project Overview

Solving the drift problem in UAV swarm operations without GPS dependency.

This project implements a comprehensive visual-inertial odometry (VIO) system for UAV swarms, combining OpenVINS, ROS2 communication architecture, and cooperative localization algorithms to achieve high-precision positioning in both simulation and real-world environments.

Tech Stack

  • Middleware: ROS2 (Humble)
  • Programming: C++17, Python
  • VIO Engine: OpenVINS
  • Simulation: Gazebo Sim(Fortress), PX4 SITL
  • Flight Control: PX4 Autopilot, MAVROS
  • Computer Vision: OpenCV, Eigen
  • Optimization: Ceres Solver, g2o
  • Visualization: RViz2

Camera-IMU Spatiotemporal Calibration using Kalibr:

  2. Intrinsic Calibration: Camera parameters with < 0.1 pixel reprojection error

Challenges & Solutions

Challenge 1: ROS2 Cross-Device Communication Latency

Problem: Network latency caused pose updates to arrive 50-100ms late.

Solution:

  • Implemented ZeroMQ-based high-frequency channel for pose data
  • Used ROS2 QoS policies for time-critical messages
  • Achieved < 10ms end-to-end latency

// Custom ZeroMQ publisher for low-latency pose streaming
auto publisher = zmq::socket_t(context, zmq::socket_style::pub);
publisher.bind("tcp://*:5555");

while (rclcpp::ok()) {
    PoseMessage msg = get_latest_pose();
    publisher.send(zmq::buffer(msg.serialize()), zmq::send_flags::none);
}

Challenge 2: IMU Noise Under Vibration

Problem: Motor vibrations caused IMU spikes, degrading VIO accuracy.

Solution:

  • Hardware: Added vibration dampening mounts
  • Software: Adaptive threshold-based outlier rejection
  • Filter: Complementary filter with vibration detection

// Adaptive IMU noise model
double get_adaptive_noise(double vibration_level) {
    return base_noise_ * (1.0 + vibration_gain_ * vibration_level);
}

// Outlier rejection
if (accel_magnitude > threshold) {
    reject_measurement();
    flag_vibration_detected();
}

Challenge 3: Computational Optimization

Problem: OpenVINS processing took 80ms per frame, limiting to 12Hz.

Solution:

  • GPU Acceleration: Ported feature tracking to CUDA
  • Multi-threading: Parallel MSCKF updates for multiple UAVs
  • Code Optimization: SIMD instructions for matrix operations

Performance Improvement:

Optimization Processing Time Frequency
Original 80ms 12Hz
+ Multi-threading 45ms 22Hz
+ GPU Acceleration 25ms 40Hz
